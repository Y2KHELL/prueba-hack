# Plantilla Proyecto Hackathon - Gestión en 5 Estándares

Proyecto provisional: **SoyCampo**  
Contexto: Hackathon Build With AI 2026 - Santa Cruz, Bolivia.  
Objetivo: gestionar el proyecto con cinco estándares simples antes de programar.

## Estructura

```text
00_CONTROL/              Guía de uso y auditoría
01_CONSTITUCION/         Problema, usuarios, valor, IA, Lean Canvas, FODA, PESTEL
02_ESPECIFICACION/       Requisitos, MVP, flujos, UI Contract, API Contract, datos
03_CLARIFICACION/        Decisiones, dudas, riesgos, supuestos y control de alcance
04_PLAN/                 Fases, arquitectura conceptual, tecnologías candidatas, Git, pruebas
05_TAREAS/               Roles, backlog, checklist, demo y pitch
06_GUIA_VISUAL/          Paleta, layout, reglas visuales y pantallas base
07_DATOS_DEMO/           JSON de noticias, campaña y lote de acopio
08_CONTRATOS/            API contract, UI routes y modelo de datos
09_REFERENCIAS_SUBIDAS/  Copia de documentos base subidos por el equipo
```

## Regla principal

No empezar a programar sin tener aprobados:
1. Constitución del proyecto.
2. Alcance MVP.
3. Flujo principal de demo.
4. UI Contract y modelo de datos mínimo.
5. Reparto de tareas por integrante.

## Decisiones importantes

- Nombre final, logo y mascota quedan pendientes.
- La tecnología final no queda cerrada en esta plantilla.
- La preferencia es Google/Firebase/Cloud/Gemini si el equipo puede implementarlo a tiempo.
- La humedad no se calcula por imagen; se registra como medición externa.
- El hardware industrial queda fuera del MVP.
- El flujo más fuerte de demo debe ser Modo Acopio por la IA visual.
